2. Attention Mechanics

Scaled Dot-Product Attention:

Attention(Q, K, V) = softmax(Q · K^T / sqrt(d_k)) · V

Dimensions:

Common bug: Forgetting to add positional encoding before the first transformer block. Without it, the model has no notion of token order and sees the input as a bag of tokens.

5. Normalization

Pre-LayerNorm (recommended):

x = x + sublayer(LayerNorm(x))

Norm BEFORE the sublayer. More stable, easier to train. Used in modern architectures (GPT-2, Llama).

Post-LayerNorm (original):

x = LayerNorm(x + sublayer(x))

Norm AFTER the residual. Harder to train deep models with. Prone to vanishing gradients.

Always use Pre-LN for decoder-only models. It's more stable, especially at initialization, and allows training deeper models without warmup tricks.
Norm TypeNormalizesWhen to Use
LayerNormOver feature dim (d_model)Standard for Transformers. Works with any seq length.
RMSNormRMS of features (no mean subtraction)Faster than LayerNorm. Used in Llama. Slightly less expressive.
BatchNormOver batch dimNOT recommended for Transformers. Varies with seq length, causes training instability.

6. Causal Masking & Causal-Carry Mismatch

Causal Mask:

def causal_mask(size):
    mask = torch.triu(torch.ones(size, size), diagonal=1).bool()
    return mask  # True = positions to mask (future tokens)

Applied as: scores = scores.masked_fill(mask, -1e9) before softmax.

Causal-Carry Mismatch (the most common bug in math transformers): Standard next-token prediction uses causal masking: token at position i can only see tokens 0...i. For math (e.g., addition), the result is computed RIGHT-to-LEFT (least significant digit first), but the model reads LEFT-to-RIGHT. This means the model must predict the result's most significant digit before seeing the least significant digits of the operands — an impossible task without carry propagation.

Tabula Rasa fix (V1.2):

  1. Reversed digits — Store operands with digits reversed: 123 + 456 becomes 321 + 654. Now the model processes LSD first, matching the right-to-left computation.
  2. Loss masking — Only compute loss on the result tokens, not on the operand tokens. This prevents the model from "wasting" capacity on reconstructing inputs.
  3. Result-first format — Place the result BEFORE the operands: =579 | 123+456. The model generates the answer token-by-token right-to-left, then sees the question.
Check: If accuracy is stuck near 0% on addition but loss decreases normally, you almost certainly have Causal-Carry Mismatch. Enable reversed digits + loss masking.

Rule of thumb: Attention contributes ~25-35% of params, FFN contributes ~55-65%. If your model is FFN-heavy, it has more memorization capacity. If attention-heavy, it has more relational capacity.

8. Debugging Checklist

When a transformer isn't learning, run through these checks in order:

  1. NaN check: torch.isnan(loss).any() — if NaN, check LR and init
  2. Loss magnitude: Is loss decreasing from initial value? If loss is flat from step 0 → check data format
  3. Initial loss sanity: For vocab_size=V, initial loss should be ~ln(V). If much lower, something is wrong.
  4. Gradient norm: Should be 0.1 - 10. Near 0 = vanishing, > 100 = exploding
  5. Overfit test: Train on 5 samples. Does the model reach 100% accuracy? If not, architecture may be broken.
  6. Shape check: Print all tensor shapes in forward pass. Verify [batch, seq, d_model] throughout.
  7. Mask check: Print attention weights. Upper-right triangle should be exactly zero.
  8. Residual check: Verify that the output from each block has the same norm scale as the input. Growing norms = instability.
  9. Data format: Print a batch of tokenized inputs and targets. Verify tokens align with expected vocabulary.
  10. Learning rate: Try lr=3e-4 as default. If loss oscillates, lower it. If loss decreases too slowly, increase it.

What This Pattern Means

Diagonal — Each token attends mostly to itself. Common in early layers of a well-trained model. Indicates token-level feature extraction.
Causal (lower-triangular) — Standard autoregressive mask. Each token can only attend to itself and previous tokens. Correct for next-token prediction.
Uniform — Token attends equally to all positions. Often seen in untrained or randomly initialized models. Should evolve into structured patterns during training.
Attend-to-first — Each token attends heavily to the first position (BOS token). Common in models trained with strong positional bias.
Attend-to-previous — Each token attends mostly to the immediately preceding token. Seen in models learning local patterns.
Attend-to-self — All attention weight is on the query token itself. May indicate collapsed attention or training issue.
Two Heads — Head 0 attends diagonally (self), Head 1 attends causally (prev + self). Healthy multi-head attention showing specialization.
Debugging tip: If all heads show uniform attention after extended training, the model may not be learning useful representations — check gradient flow and learning rate. If all attention is on the first token, the model may be ignoring positional information.

Common Attention Bugs

Self-Check: Attention Debugging Checklist

Is the causal mask applied? (upper-right should be -inf before softmax)
Are padding tokens masked? (pad token attention set to -1e9)
Is d_k correct? (d_model / n_heads — if mismatched, attention breaks silently)
Is the softmax dimension correct? (dim=-1, not dim=0)
Is the scaling factor sqrt(d_k) applied before softmax?
Are Q, K, V projections separate (no shared weight matrix by accident)?
Is the residual connection adding, not concatenating?
Is dropout applied after attention, not before?
Are attention weights normalized to sum to 1 per row? (test: all rows sum ≈ 1.0)
Is multi-head output projected back to d_model? (concat + Wo)
